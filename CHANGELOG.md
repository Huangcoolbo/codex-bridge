# 变更日志

本项目的所有重要变更都将记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，并遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [0.1.6] - 2026-03-30

### Added
- 新增 `skills/codex-bridge/SKILL.md`，将项目默认定位明确为 `Skill + Gateway` 优先的 agent 系统能力桥接器，而不是桌面客户端优先的本地演示项目。

### Changed
- 调整 Android 桌面端工作流；无线配对从“二维码导入/扫描”收敛为“手机显示配对地址与配对码，客户端执行 pair / 自动查找连接地址 / 自动 connect”的真实 ADB 流程。
- 重构 Android 页结构，将无线配对、USB 设备、目标选择和最近设备重连拆成更清楚的独立区域，并统一了对应的中英文文案。
- 调整顶部状态栏与彩蛋反馈机制；状态颜色、紧凑状态条和顶部提示文案现在会围绕真实任务阶段组织，而不再只是静态状态显示。
- 调整全局按钮和列表型控件样式，提升浅色按钮边框、阴影和 hover/active 对比度，使 Windows、Android 和其他栏目保持一致的可点击控件语义。

### Fixed
- 修复 Android `adb connect` 在某些情况下 `stdout` 已明确写出 `cannot connect`、但包装结果仍被判为成功的问题；现在会结合输出内容将其归一为失败。
- 修复 Android 无线工作流中“配对成功但整条自动流程仍被粗暴显示为配对失败”的状态表达问题；现在会分阶段显示“配对中 / 查找连接地址中 / 连接中”。
- 修复 Android 无线与 USB 目标在桌面端共用同一保存路径时的语义混乱问题；现在会按目标类型分别组织目标值和重连行为。

## [0.1.5] - 2026-03-27

### Added
- 新增桌面客户端更新后的自动安装流；下载完成后会自动启动新安装包并退出当前客户端，减少手动打开安装包的步骤。

### Changed
- 调整桌面客户端更新下载策略；更新安装包统一下载到应用自管的 `userData/updates` 目录，不再依赖系统下载目录，避免异常路径导致下载失败。

### Fixed
- 修复部分机器上 `app.getPath("downloads")` 返回盘符根目录（如 `D:\`）导致更新下载阶段报 `EPERM: operation not permitted, mkdir 'D:\'` 的问题。
- 新增客户端启动时对旧更新安装包的清理逻辑，避免更新目录中长期残留旧版本安装文件。

## [0.1.4] - 2026-03-27

### Changed
- 重做桌面客户端顶部栏交互；当中间工作区滚动时，顶部栏会从居中状态过渡到左右分裂式布局，品牌与状态收拢到左侧，操作按钮稳定停留在右侧。
- 调整顶部栏的悬浮高度、宽度节奏和中间动态文案，使其更接近桌面工作台而不是普通网页导航条。

### Fixed
- 进一步隐藏启动时的后台 bootstrap PowerShell 窗口，减少用户在打开软件时看到终端弹出的情况。

## [0.1.3] - 2026-03-27

### Added
- 新增基于 GitHub Release 的桌面端版本检查能力；客户端启动后可自动检查最新发布版本，并在发现新版本时弹出本地更新提示。
- 新增桌面端更新下载流程；用户可直接从客户端下载最新安装包，并在下载完成后直接打开安装包。
- 新增系统托盘右键菜单，提供版本查看、手动检查更新、查看发布页和退出入口。

### Changed
- 新增 Windows NSIS 安装版打包配置，并将安装版作为默认 `dist:win` 发布目标，以降低日常启动时的单文件解包开销。
- 将项目图标打入安装包资源目录，供系统托盘和桌面端更新入口复用。

## [0.1.2] - 2026-03-27

### Changed
- 优化打包版桌面客户端的运行时同步策略；不再每次启动都全量复制 runtime 资源，改为仅在版本变化或运行目录缺失时同步，缩短重复启动时的等待时间。
- 调整 Windows 主任务流为“先发现主机、再探测、成功后可保存”，不再要求用户必须先保存配置才能执行探测。
- 收敛客户端操作状态显示，移除左侧与右侧重复状态块，仅保留顶部状态栏作为统一状态入口。

### Fixed
- 修复顶部状态栏只显示进行中、不明确反映探测成功和保存成功的问题；现在会直接显示成功或失败结果。
- 修复 Windows 未保存连接草稿无法直接探测的问题；新增受控草稿探测链路，不写入注册表也可完成 probe。

## [0.1.1] - 2026-03-27

### Fixed
- 修复打包版桌面客户端只检查 `.venv` 是否存在、却不会校验 Python 依赖是否完整的问题；现在会检查 `paramiko` 和项目包是否可导入，并在不完整时自动重建与补装依赖。
- 修复打包版 localhost 自管 SSH key 缺少初始化与授权链路的问题；新增受控脚本，为客户端运行目录下的 `data/ssh/localhost_ed25519` 生成和授权本机 SSH key，减少首次使用打包版时本机探测失败的问题。

## [0.1.0] - 2026-03-27

### Added
- 新增 Electron + React + TypeScript 桌面客户端，提供 Windows、Android、Linux 三个平台入口与桌面化工作台界面。
- 新增本地 HTTP agent gateway，供 Codex / agent 通过正式接口执行 target 管理、probe、命令执行和结果读取。
- 新增 Windows target 管理与会话能力，包括 target 列表、详情、当前 target、probe、命令草稿、执行结果和最近结果接口。
- 新增 Android gateway 只读接口，包括设备列表、设备信息、安全 shell、文件列表、文件读取、文件拉取和当前设备会话。
- 新增 Android gateway 第一批受控写接口，包括 `files/mkdir`、`files/write`、`files/push`，并对路径白名单、写入模式、内容大小和覆盖行为做限制。
- 新增 Android USB 真机接入能力，并完成已连接设备的 target 保存与执行链路验证。
- 新增独立网关文档 [AGENT_GATEWAY.md](./AGENT_GATEWAY.md)，整理可供外部调用的 HTTP API、字段映射、安全边界和推荐调用流程。
- 新增 `docs/GETTING_STARTED*` 和 `docs/DEVELOPMENT*` 文档，将仓库首页、快速上手和开发说明分层。
- 新增 Windows 单文件 portable 打包链路，可生成直接运行的桌面可执行文件。
- 新增运行时路径抽象与启动 bootstrap，使打包后的客户端能够自动准备 Python、ADB、bridge 资源和用户数据目录。
- 新增应用图标资源与 Windows 打包图标配置。

### Changed
- 重构 README，使仓库首页从开发说明书调整为面向用户的产品介绍页，并采用更适合 GitHub 首页阅读的中英文结构。
- 将 GitHub 默认首页调整为中文版本，并保留英文首页入口。
- 调整 README 顶部徽章，增加平台、客户端、gateway、文档、star 和 fork 信息。
- 将桌面客户端的发布名统一为 `Codex.Bridge`，并输出正式 portable 文件名 `Codex.Bridge-v0.1.0.exe`。
- 将 Windows 默认密钥策略调整为优先使用项目内自管密钥，而不是默认依赖当前用户目录下的 `~/.ssh`。
- 将桌面客户端的当前配置、当前 target 和当前 Android 设备状态从 UI 层向主进程能力层收敛，减少 UI 状态与网关状态不一致的问题。

### Fixed
- 修复 GitHub 上 README 与 docs 文档链接使用本地绝对路径导致的失效问题，统一改为仓库相对路径。
- 修复默认中文 README 入口结构冗余问题，去掉多余的中文别名页。
- 修复桌面客户端 preload 路径错误导致的 `window.bridgeDesktop` 注入失败问题。
- 修复平台切换时顶部配置选择区残留上一平台选中状态的问题，确保 Windows / Android / Linux 配置状态按平台隔离。
- 修复客户端任务状态显示中失败态仍带进行中文案的问题，使失败状态与最近任务结果一致。
- 修复 Android `files/push` 在 `overwrite=false` 时的远端存在性判断问题。
- 修复 Android shell 脚本执行包装带来的输出噪音和 shell 兼容性问题。
- 修复桌面客户端在窗口缩小时中间主内容区收缩不足、左右栏跟随整页滚动等布局问题。
- 修复打包后客户端仍依赖开发目录 `process.cwd()` 的路径假设问题，使运行时资源、脚本和数据目录在发布环境中可用。
